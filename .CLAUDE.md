# gatekeeper-sut — Claude Coding Guide

## What This Repo Is

Platform-agnostic shell-based Docker image build gate. Inspired by Docker Hub's Automated Repository
Tests. One script (`sut-runner.sh`) fetched by CI, runs `*.test.yaml` SUT tests, blocks docker push
on failure.

## Architecture (one-liner)

`sut-runner.sh` discovers `*.test.yaml` → runs each via Docker Compose with an injected isolated
network → exits non-zero on any failure → CI blocks the push.

## Core Coding Principles

- **Pure bash** — no external dependencies beyond docker + standard unix tools
- **Bash 3.2+ compatible** — macOS ships bash 3.2; avoid bash 4+ features (`declare -A`, `mapfile`)
- **POSIX-safe** — use BSD-compatible `sed`, `date +%s`; test on both macOS and Linux
- **shellcheck clean** — run `shellcheck` on every bash file before committing; fix all warnings
- **Minimal** — one script, no frameworks, no config files beyond the test yamls
- **Fail loudly** — `set -euo pipefail`; explicit exit codes; log everything to stderr on error
- **Always clean up** — `trap cleanup EXIT` scoped to each test subshell; network + compose down

## Key Files

| File | Purpose |
|------|---------|
| `sut-runner.sh` | Main runner — the only deliverable CI consumes |
| `examples/hooks/pre_push/` | Reference test fixtures |
| `docs/plans/` | Design docs and POC notes |

## Conventions

- Test files: `*.test.yaml` / `*.test.yml` with a required `sut` service
- Test dir in app repos: `hooks/pre_push/`
- Network names: `sut_net_<testname>_<pid>_<timestamp>` — unique, safe for parallel runs
- Project names: `sut_<testname>_<pid>_<timestamp>`
