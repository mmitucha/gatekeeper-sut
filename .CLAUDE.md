# gatekeeper-sut

## Project Overview
A platform-agnostic shell-based SUT (System Under Test) runner that enforces Docker image quality
by running containerized tests before allowing a Docker push. Inspired by Docker Hub's
"Automated Repository Tests" concept — portable to any CI/CD platform.

## Core Concept
- Test definitions (`*.test.yaml`) live **inside the tested app's repo** under `hooks/pre_push/`
- The runner (`sut-runner.sh`) lives **in this repo** and is fetched/called by CI pipelines
- Blocking Docker push without passing tests is the primary goal (Build Gate / PR validation)

## Repository Structure
```
gatekeeper-sut/
├── sut-runner.sh           # Main runner — entry point for CI
├── CLAUDE.md
├── README.md
└── examples/
    └── hooks/
        └── pre_push/
            ├── smoke.test.yaml
            └── smoke_test.sh
```

## Test Convention (in the app repo)
```
app-repo/
└── hooks/
    └── pre_push/
        ├── *.test.yaml     # docker compose test definitions (sut service required)
        └── *.sh            # test scripts referenced by sut containers
```

- Test files follow Docker Hub convention: `*.test.yaml` with a `sut` service
- `sut` container exit code drives pass/fail via `--exit-code-from sut`

## Runner Behavior
- Discovers all `*.test.yaml` in the provided `--tests-dir` path
- Each test runs on its own **temporary isolated Docker network** (created by runner, not the yaml)
- Network is injected via compose override (`-f`) — app's test.yaml needs no network definition
- Network + project name use unique suffix (`<name>_<pid>_<timestamp>`) — safe for parallel agents
- Cleanup always runs via `trap cleanup EXIT` — on pass, fail, or signal
- Handles both `docker compose` (v2) and `docker-compose` (v1) — detected once at startup
- Exits non-zero if any test fails → blocks the push in CI

## CI Integration Pattern
Runner is fetched and executed by CI. Docker push job depends on it.

### Azure DevOps
```yaml
- job: SUT_Tests
  steps:
    - script: |
        curl -fsSL https://raw.githubusercontent.com/<org>/gatekeeper-sut/main/sut-runner.sh \
          -o /tmp/sut-runner.sh
        chmod +x /tmp/sut-runner.sh
        /tmp/sut-runner.sh --tests-dir ./hooks/pre_push

- job: DockerPush
  dependsOn: SUT_Tests
  condition: succeeded()
```

### GitHub Actions
```yaml
jobs:
  sut-tests:
    steps:
      - run: |
          curl -fsSL https://raw.githubusercontent.com/<org>/gatekeeper-sut/main/sut-runner.sh \
            -o /tmp/sut-runner.sh
          chmod +x /tmp/sut-runner.sh
          /tmp/sut-runner.sh --tests-dir ./hooks/pre_push

  docker-push:
    needs: sut-tests
```

## Key Technical Decisions
- **Pure bash** — no dependencies beyond docker and standard unix tools
- **Network override via second `-f`** — forces `default` network to runner-created external network
  without requiring app's test.yaml to declare any network
- **Subshell per test** — scopes `trap` cleanup to each test independently
- **Sequential test execution** — keep simple, parallel adds complexity without early benefit
- **`--abort-on-container-exit` + `--exit-code-from sut`** — compose exits when sut finishes
